Core System Specifications

  • Event-Driven Microservices: Asynchronous processing via Azure Service Bus.

Pros of this Architecture

  • Total Decoupling: Heavy 20-second LLM generations never block the FastAPI threads.
  • Resilience: If LangGraph crashes, Azure Service Bus automatically requeues the message.
  • Auto-Scaling: Azure Container Apps automatically scale LangGraph instances up based on Service Bus queue depth (and down to zero when idle).

Trade-offs / Cons

  • Eventual Consistency: The Frontend cannot wait for an HTTP response; it must poll DB or use WebSockets.
  • Distributed Debugging: A single failure requires cross-service log tracing.

CI/CD & Azure Deployment

Automated pipeline from Commit to Cloud

GitHub Actions (CI/CD)

Every push to `main` triggers a fully automated GitHub Actions workflow that lints the Python/React code, runs PyTest (unit tests), builds Docker containers, and pushes them to the Azure registry.

Azure Static Web Apps & Container Apps

The React frontend is globally hosted on Azure Static Web Apps. The FastAPI backend and LangGraph workers run in Azure Container Apps (ACA), automatically scaling from zero to 100+ instances based on Azure Service Bus queue depth.
